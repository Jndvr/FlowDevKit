FlowDevKit

Privacy Policy

Last updated: March 2026  ·  Author: Jan Düver

Short version: FlowDevKit collects no personal data, sends nothing to external servers, and stores only your local UI preferences. Everything runs in your browser.

1. What FlowDevKit Does

FlowDevKit is a browser extension for Microsoft Power Automate developers. It reads the flow definition from the Power Automate API using your existing browser session and presents it through a local developer toolbox (copy JSON, inspect expressions, analyze flow quality, view run errors, and more).

2. Data We Do Not Collect

FlowDevKit does not:

  • Collect, store, or transmit your flow data to any server
  • Collect personal information (name, email, account details)
  • Track usage, analytics, or telemetry of any kind
  • Store authentication tokens, passwords, or session cookies beyond your local browser session
  • Communicate with any server other than official Microsoft Power Platform APIs

3. What Stays on Your Device

The only data FlowDevKit stores locally (via chrome.storage.local) is your UI preferences:

  • Selected API region (e.g. EMEA, US)
  • Export format preference (Definition only / Full export)
  • Color theme (light / dark)
  • Custom API host (if entered)

These preferences never leave your device and can be cleared at any time by removing the extension.

4. Permissions Explained

PermissionWhy it's needed
activeTab Reads the URL of your current tab to extract the environment ID and flow ID required to call the Power Automate API.
tabs Scans tab frames to locate the correct Power Automate frame URL, since the designer loads inside an iframe.
scripting Executes API fetch calls in the Power Automate tab context so requests carry your existing session authentication.
clipboardWrite Copies flow JSON or selected actions to your clipboard when you click Copy.
clipboardRead Reads clipboard content when you use the Paste Actions feature to inject previously copied actions into a flow.
cookies Reads your Power Automate session cookie to authenticate API calls made from the background service worker. No cookie data is stored or transmitted outside your browser.
webNavigation Detects tab navigation so the extension can refresh its flow context and warn you when the context may be stale.
sidePanel Enables the extension to open as a docked side panel alongside the Power Automate designer.
storage Saves your UI preferences (region, format, theme) locally so they persist across sessions.

5. Third-Party Services

FlowDevKit only communicates with official Microsoft Power Platform endpoints:

  • *.api.flow.microsoft.com — Power Automate REST API
  • make.powerautomate.com / make.powerapps.com — Maker portal
  • *.environment.api.powerplatform.com — Power Platform environment API
  • login.microsoftonline.com — Microsoft authentication
  • api.bap.microsoft.com — Business Application Platform API

No data is sent to the extension author or any other third party.

7. Changes to This Policy

If this policy is updated, the "Last updated" date at the top of this page will change. Significant changes will also be noted in the extension's release notes on GitHub.
